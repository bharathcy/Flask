Test Case to Test Script Generation

This tool takes the test case file as input and generates test script aligned with the initial and expected values.




Procedure:

  • The test case file should be in excel format (.xlsx)
  • Test case should be written as per syntax provided in SWVCP otherwise tool will raise error.
  • Test case file should be placed in the folder where TC to TS conversion tool (TestScript_Final.py) is present.
  • Double click on the TestScript_Final.py executable and below window will open
  • Press enter and select the test case file from the directory.
  • Once file is selected, prompt will ask to enter environment name. Enter environment name as per syntax.
  • Next prompt will be to enter the Unit under test name. Enter the same and press enter.
  • Next prompt will be to enter the unit test file name which is nothing other than the unit under test. Enter the same and press enter.
  • Enter the name of function under test when it prompts to enter sub program name.
  • Select template file (Template_File.txt) provided in the directory when prompt comes for selecting template file.
  • Finally, when prompt comes for entering the output file name, enter the test script name required as per the syntax and press enter to generate the test script.
  • Generated test script can be found in “Generated TS” folder inside the same directory.
  • In case generated script is corrupted or contains less number of test cases, re-generate the script by opening the tool via command prompt. This way, one can find out the error in test case file, fix and re-generate.



Output:

Output file *.tst will be generated
